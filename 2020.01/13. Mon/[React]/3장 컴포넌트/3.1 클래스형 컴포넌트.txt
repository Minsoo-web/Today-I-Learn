컴포넌트를 선언하는 방식은 두 가지입니다. 
1. 함수형 컴포넌트 
2. 클래스형 컴포넌트

클래스형 컴포넌트는 다음과 같이 구성됩니다. 

    import React,{ Component} from 'react';

    class App extends Component{
        render(){
            const name = 'react';
            return(
                <div className="react">
                {name}
                </div>
            );
        }
    }

    export default App; 
    ** 클래스 : rcc 
       함수형 : rsc 

함수형 컴포넌트와 클래스형 컴포넌트의 차이점은 
클래스형 컴포넌트의 경우 이후 배울 state기능 및 라이프사이클 기능을 사용할 수 있다는 점과 
                            임의 메서드를 정의할 수 있다는 점입니다. 

클래스형 컴포넌트에서는 render함수가 꼭 있어야 하고 
그 안에서 보여주어야 할 JSX를 반환해주어야 합니다. 

함수형 컴포넌트의 장점 : 
    우선 클래스형 컴포넌트보다 선언이 훨씬 편함
    메모리 자원도 클래스보다 덜 사용 (미미한 수준)

함수형 컴포넌트의 단점은 state와 라이플사이클 API 사용이 불가능하다는 점인데 
v16.8 업데이트 이후 Hooks라는 기능이 도입되며 해결

리액트 공식 메뉴얼에서는 컴포넌트를 새로 작성할 때 함수형 컴포넌트와 Hooks를 사용하도록 권장하고 있습니다. 
하지만 그렇다고 해서 클래스형 컴포넌트가 사라지는 것은 아니므로 클래스형 컴포넌트의 기능은 꼭 알아두어야 합니다.
