웹 애플리케이션을 만들다 보면 코드에서 같은 형태의 코드가 반복되는 경우를 볼 수 있습니다. 

코드가 복잡해지면 복잡해질 수록 이러한 반복은 낭비입니다. 
또 보여주어야 할 데이터가 유동적이라면 이런 코드로는 절대 관리할 수 없습니다. 

이 장에서는 리액트 프로젝트에서 반복적인 내용을 효율적으로 보여 주고 관리하는 방법을 알아보겠습니다. 

6.1.1 map 문법 
arr.map(callback,[thisArg])

callback : 새로운 배열의 요소를 생성하는 함수로 파라미터는 다음 세 가지입니다. 
    - currentValue : 현재 처리하고 있는 요소 
    - index : 현재 처리하고 있는 요소의 index 값 
    - array : 현재 처리하고 있는 원본 배열 
thisArg(선택) : callback 함수 내부에서 사용할 this 레퍼런스
